# AutoPas Styleguide

## Introduction

This style guide outlines the coding conventions for our C++ library AutoPas

## Key Principles

*   **Readability:** Code should be easy to understand for all and new team members.
*   **Maintainability:** Code should be easy to modify and extend.
*   **Consistency:** Adhering to a consistent style across all projects
    improves collaboration and reduces errors.
*   **Performance:** While readability is paramount, code should be efficient.

### Modern

* Use modern C++20 code principles and concepts whenever possible

### Performance

* High performance is crucial and may be prioritized over readability

### Documentation

* Every function should be properly documented
* Complex code parts should have appropriate comments
